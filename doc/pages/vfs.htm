Zicronix Virtual File System

Note: Zicronix's VFS is still under development, so the factual accuracy of this document is disputed.

The Virtual File System in Zicronix is pretty simple, the application makes a request through applib which issues an interrupt and tells
the kernel about a request, the kernel handles the request by grouping it into:
1. Is it a R/W operation to a device?
2. Or is it a typical function call?
For case(2) the kernel will simply call the appropriate handler and exit, for case(1) the kernel will make a request to the HAL through
the VFS, the HAL will find the appropriate device driver, and will issue a request to the driver, the driver will have all rights
to access the device it's give and will write/read to the device.
